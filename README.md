# WeakUP 早起考研打卡应用

## 项目介绍 (Project Introduction)
WeakUP 是一个专为“考研早起”定制的 30 天打卡激励纯前端网页应用。应用设计极简却充满细节，融合了极致的日出渐变（Sunrise Gradients）和玻璃拟物化设计（Glassmorphism）。
所有数据均安全保存在本地（LocalStorage），这意味着无需配置任何后端服务器和数据库，只需静态托管即可即开即用！

### 核心功能
- **早起强制检验**：系统会对照设备时间，严格限制每日打卡时间（必须在 06:40 AM 之前），有效克服早起惰性。
- **打卡日签生成器**：借助相机拍摄起床照片后，应用会自动拼接时间戳、天数、累计奖励金，并自动抽取考研专属励志语录，生成一张能够极速下载分享的打卡日签海报。
- **30天历程星图**：内置记录日历墙，30天内的坚持和中断天数一目了然，清晰记录追梦过程中的每一个早晨。

## 快速免费部署指南 (开启 GitHub Pages 服务)

既然项目代码已经存在于 GitHub 仓库中，这里为您提供如何直接一键开启免费在线访问体验的步骤（不需要接触任何系统代码，只需要点击几次鼠标）：

1. **进入仓库设置界面**：
   在网页端进入该项目的 GitHub 仓库页面，点击代码区域上方最右侧的 **Settings (齿轮图标)**。

2. **找到 Pages 服务**：
   在左侧的长条菜单中向下滑动，找到 **Pages** (通常在最下方一列偏上的位置)，进入服务配置中心。

3. **设置发布源**：
   在打开的页面中，找到 **Build and deployment** 区块。
   在其下方的 **Source** 下拉菜单中，确认选择了 **Deploy from a branch**。

4. **确认发布分支**：
   在紧接着下方的 **Branch** 选择框里：
   - 将 `None` 切换为 `main`（或者 `master`）。
   - 确认右侧的文件夹选择为根目录 `/(root)`。
   - 点击右侧的 **Save** 按钮进行保存。

5. **获取专属链接**：
   保存后等待大概 1~2 分钟，刷新当前页面，页面顶部会刷新出一条类似如下格式的成功提示语以及可点击的链接：
   > *"Your site is live at https://您的用户名.github.io/WeakUP/"*
